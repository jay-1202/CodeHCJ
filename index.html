HTML

=> Basic Structure Tags

  • <html> tag

    The <html> element is used as a container for all of the HTML of an entire document.

  • <head> tag

    The <head> element contains information about an HTML document that is used by browsers and web crawlers but is not displayed to website visitors.

  • <title> tag

    The <title> element is a required HTML element used to assign a title to an HTML document.

  • <body> tag

    The <body> element contains the entire content of a webpage. It must be the second element inside of the parent html element, following only the <head> element.


=> Text and Formatting

  • <h1> to <h6> tag

    Heading tags define section titles, with <h1> being the most important and <h6> the least.

  • <p> tag

    Defines a paragraph of text, automatically adding spacing before and after.

  • <br> tag

    Inserts a single line break without starting a new paragraph.

  • <hr> tag

    Creates a horizontal rule, often used to separate content sections.

  • <strong> tag

    Emphasizes text with semantic importance, typically rendered bold.

  • <em> tag

    Emphasizes text with stress, typically rendered italic.

  • <b>/<i> tag

    Applies bold (<b>) or italic (<i>) styling without semantic emphasis.

  • <mark> tag

    Highlights text with a yellow background to indicate relevance.

  • <small> tag

    Displays text in a smaller font size, often for disclaimers or fine print.

  • <sub>/<sup> tag

    Formats text as subscript (<sub>) or superscript (<sup>), useful in formulas.


=> Links and Navigation

  • <a> tag

    Defines a hyperlink, allowing users to navigate to another page, section, or external site.

  • <nav> tag

    Represents a section of navigation links, typically used for menus or site navigation.

  • <ul>/<ol> tag

    Creates unordered (<ul>) or ordered (<ol>) lists for grouping related items.

  • <li> tag

    Defines a list item inside <ul> or <ol>, representing individual entries.


=> Media and Embeds

  • <img> tag

    Embeds an image into the page using the src attribute to specify the file path.

  • <video>/<audio> tag

    Embeds video or audio content with optional controls, autoplay, and loop attributes.

  • <source> tag

    Specifies multiple media sources for <video> or <audio>, allowing fallback formats.

  • <iframe> tag

    Embeds another HTML page or external content like videos, maps, or widgets.

  • <canvas> tag

    Provides a drawable region for rendering graphics via JavaScript.

  • <svg> tag

    Defines scalable vector graphics directly in HTML, ideal for icons and illustrations.


=> Layout and Semantic

  • <div> tag

    A generic container used to group elements for styling or scripting purposes.

  • <span> tag

    An inline container used to style or group text without affecting layout.

  • <section> tag

    Defines a standalone section of content, often with its own heading.

  • <article> tag

    Represents self-contained content like blog posts, news articles, or forum entries.

  • <aside> tag

    Contains content tangentially related to the main content, like sidebars or callouts.

  • <header>/<footer> tag

    Defines introductory (<header>) or closing (<footer>) content for a section or page.

  • <main> tag

    Specifies the primary content of a document, excluding navigation, sidebars, and footers.


=> Metadata and Scripts

  • <meta> tag

    Provides metadata about the document, such as character encoding, author, and viewport settings.

  • <link> tag

    Defines relationships to external resources like stylesheets or icons.

  • <style> tag

    Contains internal CSS rules that apply styling directly within the HTML document.

  • <script> tag

    Embeds or links to JavaScript code for dynamic behavior and interactivity.

  • <noscript> tag

    Displays fallback content for users whose browsers do not support JavaScript.


=> Forms and Inputs

  • <form> tag

    Creates an HTML form for user input, with attributes for action, method, and encoding.

  • <input> tag

    Defines various types of input fields like text, checkbox, radio, and more.

  • <textarea> tag

    Creates a multi-line text input field for longer user responses.

  • <button> tag

    Defines a clickable button, which can submit forms or trigger scripts.

  • <select>/<option> tag

    Creates a dropdown menu (<select>) with selectable choices (<option>).

  • <label> tag

    Associates text labels with form controls to improve accessibility and usability.

  • <fieldset>/<legend> tag

    Groups related form elements (<fieldset>) and provides a caption (<legend>).


=> Interactive and Advanced

  • <details>/<summary> tag

    Creates a collapsible section (<details>) with a clickable label (<summary>) to toggle visibility.

  • <dialog> tag

    Defines a modal or popup dialog box that can be opened and closed with JavaScript.

  • <template> tag

    Holds client-side content that is not rendered when the page loads but can be activated later via JavaScript.

  • <slot> tag

    Used in Web Components to define placeholder content that can be filled by external markup.


CSS

=> Box Model Properties

  • width property

    Defines the horizontal size of an element's content box.

  • height property

    Defines the vertical size of an element's content box.

  • padding property

    Creates space between the content and the element's border.

  • margin property

    Creates space outside the element's border, separating it from other elements.

  • border property

    Defines the thickness, style, and color of the element's border.

  • box-sizing property

    Controls whether padding and border are included in the element's total width and height.


=> Typography Properties

  • font-family property

    Specifies the typeface used for text content.

  • font-size property

    Sets the size of the text.

  • font-weight property

    Controls the thickness of the font (e.g., normal, bold, 400, 700).

  • line-height property

    Sets the vertical spacing between lines of text.

  • text-align property

    Aligns text horizontally (left, right, center, justify).

  • color property

    Sets the color of the text.


=> Layout & Positioning

  • display property

    Controls the layout behavior of an element (e.g. block, inline, flex, grid).

  • position property

    Specifies how an element is positioned in the document (static, relative, absolute, fixed, sticky).

  • top/right/bottom/left property

    Offsets a positioned element from its reference edge.

  • z-index property

    Controls the stack order of overlapping elements along the z-axis.

  • overflow property

    Determines what happens when content overflows its container (visible, hidden, scroll, auto).

  • visibility property

    Controls whether an element is visible or hidden without affecting layout.


=> Flexbox & Grid

  • display: flex/grid property

    Enables flex or grid layout mode for arranging child elements.

  • justify-content property

    Aligns items along the main axis in a flex or grid container.

  • align-items property

    Aligns items along the cross axis in a flex or grid container.

  • flex-grow/flex-shrink property

    Controls how flex items grow or shrink relative to the container and each other.

  • grid-template-columns/rows property

    Defines the structure of rows and columns in a grid layout.

  • gap property

    Sets spacing between rows and columns in flex or grid layouts.


=> Visual Effects

  • background property

    Sets the background color, image, or gradient of an element.

  • box-shadow property

    Applies shadow effects around an element's frame.

  • opacity property

    Controls the transparency level of an element.

  • filter property

    Applies graphical effects like blur, brightness, or contrast to an element.

  • border-radius property

    Rounds the corners of an element's border box.


=> Transitions & Animation

  • transition property

    Defines how property changes occur over time (e.g., duration, timing function).

  • transform property

    Applies 2D or 3D transformations like rotate, scale, skew, or translate.

  • animation property

    Controls keyframe-based animations including duration, delay, and iteration.

  • @keyframes property

    Defines the stages of an animation by specifying property values at key points.

  • will-change property

    Hints to the browser which properties are likely to change, optimizing performance.



JS

=> Core Syntax & Variables

  • JavaScript Syntax

    Defines how JavaScript code is structured and executed. Statements typically end with semicolons and are case-sensitive.

  • Comments

    Used to explain code or disable execution. Single-line uses //, multi-line uses /* */.

  • Variables

    Used to store data. let and const are block-scoped and preferred over var.

  • Data Types

    Includes primitives like String, Number, Boolean, and reference types like Object and Array.

  • Type Conversion

    Converts values between types using functions like Number(), String(), and Boolean().


=> Operators & Expressions

  • Operators

    Used to perform operations on values and variables. Includes arithmetic, assignment, comparison, and logical operators.

  • Expressions

    Combinations of values, variables, and operators that produce a result. Example: 2 + 3 is an expression.


=> Control Flow & Loops

  • if / else

    Conditional statements that execute code blocks based on true or false conditions.

  • switch / case

    Used to test multiple values against a single expression. Each case runs if matched.

  • Loops

    Repeat code using for, while, or do...while until a condition is false.

  • break / continue

    break exits a loop early; continue skips to the next iteration.


=> Functions & Scope

  • Function Declaration

    Defines a reusable block of code using the function keyword.

  • Function Expression

    Assigns a function to a variable. Can be anonymous or named.

  • Arrow Functions

    Concise syntax for writing functions using =>. Inherits this from parent scope.

  • Parameters & Arguments

    Parameters are placeholders in function definitions; arguments are actual values passed during calls.

  • Return Statement

    Specifies the value a function returns to its caller.

  • Default & Rest Parameters

    Default parameters provide fallback values; rest parameters collect multiple arguments into an array.

  • Scope & Hoisting

    Scope defines variable visibility. Hoisting moves declarations to the top of their scope during execution.


=> Objects & Arrays

  • Object Basics

    Objects store key-value pairs and represent structured data. Defined using curly braces {}.

  • Properties & Methods

    Properties hold values; methods are functions attached to objects. Accessed via dot or bracket notation.

  • this Keyword

    Refers to the object that is executing the current function. Behavior depends on context.

  • Constructors

    Functions used to create multiple object instances with shared structure.

  • Prototypes

    Enable inheritance in JavaScript. Objects can inherit properties and methods from their prototype chain.

  • Creating Arrays

    Arrays hold ordered collections of values. Defined using square brackets [].

  • Array Methods

    Common methods include push, pop, map, filter, and reduce.

  • Array Iteration

    Loop through arrays using for...of, forEach, or traditional for loops.


=> Strings, Numbers & Math

  • String Methods

    Manipulate text using methods like slice, replace, toUpperCase, and split.

  • Template Literals

    Use backticks ` ` and ${} for embedded expressions and multi-line strings.

  • Number Methods

    Format and convert numbers using toFixed, parseInt, and parseFloat.

  • Math Object

    Provides mathematical functions like Math.round, Math.random, and Math.max.


=> Dates & Timers

  • Date Object

    Represents dates and times. Created with new Date().

  • Date Methods

    Access and modify date components using getDate, getMonth, setFullYear, etc.

  • Timers

    Use setTimeout and setInterval to delay or repeat code execution.


=> DOM & Events

  • Selecting Elements

    Access HTML elements using getElementById, querySelector, and related methods.

  • Changing Content & Styles

    Modify element content with innerText, innerHTML, and style with style or classList.

  • Creating / Removing Elements

    Use createElement, appendChild, and removeChild to manage DOM nodes.

  • Events & Listeners

    Respond to user actions using addEventListener and event handlers like onclick.


=> Browser APIs (BOM)

  • Window Object

    Represents the browser window and provides access to global properties, methods, and events.

  • Navigator

    Contains information about the browser, such as user agent, language, and online status.

  • Location

    Provides the current URL and allows navigation or redirection using location.href.

  • Alerts / Prompts / Confirms

    Display simple dialog boxes using alert(), prompt(), and confirm().

  • Local & Session Storage

    Store key-value data in the browser using localStorage and sessionStorage.


=> ES6+ Features

  • let & const

    Modern variable declarations. let is block-scoped and reassignable; const is block-scoped and constant.

  • Arrow Functions

    Concise syntax for functions using =>. Lexically binds this.

  • Destructuring

    Extract values from arrays or objects into distinct variables.

  • Spread & Rest

    ... Spread expands elements; Rest collects multiple elements into one.

  • Modules

    Use import and export to organize code across files.

  • Classes

    Blueprints for creating objects using class syntax with constructors and methods.

  • Promises

    Handle asynchronous operations with then() and catch().

  • Async / Await

    Simplifies asynchronous code by allowing it to look synchronous. Works with Promises.


=> Error Handling

  • try / catch / finally

    Used to catch and handle errors gracefully. finally runs regardless of outcome.

  • throw

    Manually trigger an error using throw with a message or object.

  • debugger

    Pauses code execution in debugging tools for inspection.


=> JSON & Fetch

  • JSON Basics

    JavaScript Object Notation is a lightweight data format. Use JSON.parse() and JSON.stringify().

  • Fetch API

    Used to make network requests. Returns a Promise and supports chaining.

  • Handling Responses

    Use .then() to process response data and .catch() for errors.


=> Advanced Concepts

  • Closures

    Functions that retain access to their outer scope even after the outer function has returned.

  • Callbacks

    Functions passed as arguments to other functions, often used for asynchronous operations.

  • Event Loop

    Handles asynchronous behavior by managing the call stack and task queue.

  • Execution Context

    Defines the environment in which JavaScript code is evaluated and executed.

  • this Binding

    Determines the value of this based on how a function is called.

  • Prototype Inheritance

    Allows objects to inherit properties and methods from other objects via the prototype chain.
